Nice work. The takeaway from Chapter 3: never paste production credentials, payment data, or personal data into a public LLM. Mask it, swap in synthetic/test data, or use an enterprise model with a data-processing agreement. Confident-looking output is no excuse for leaking PII — the privacy obligation sits with the tester, not the tool.

What the scanner looks for: email addresses, payment card numbers (PAN), CVV codes, IBAN/account numbers, phone numbers, API keys / bearer tokens, internal hostnames & IP addresses, and named individuals. Replacing the value with any placeholder removes the flag.
